## Stack Decisions

| Concern | Choice | Rationale |
|---|---|---|
| Runtime | Bun | Native TypeScript, fast startup, built-in test runner compatibility |
| Framework | Fastify | Structured plugin system, JSON schema validation, Pino logging built-in |
| Chain interaction | viem | Type-safe, modern, Morpho SDK compatible, native EIP-1559 support |
| Morpho SDK | `@morpho-org/blue-sdk` + `@morpho-org/blue-sdk-viem` | Official Morpho TypeScript SDK for reading market/vault state |
| Scheduler | croner | Lightweight cron for Bun, no Redis dependency (unlike BullMQ) |
| Testing | Vitest + Anvil | Vitest for unit tests, Anvil for forked mainnet integration tests |
| Config validation | zod | Runtime schema validation, great TypeScript inference |
| Logging | Pino | Fastify-native, structured JSON, zero-overhead in production |
| Telegram | Direct Bot API via `fetch` | No SDK dependency; single `sendMessage` endpoint is all we need |
| Deployment script | Foundry (forge) | Industry standard for Solidity scripting, fork-test support |
| Database | None (v1) | On-chain is the data source; DB for analytics is a v2 candidate |

## Architecture Patterns

> Full diagrams and rationale: [docs/architecture.md](docs/architecture.md)

### Service-Oriented Architecture

```
Fastify (HTTP + Plugins)
  ‚îÇ
  ‚îú‚îÄ‚îÄ scheduler plugin  ‚Üí cron trigger
  ‚îú‚îÄ‚îÄ health plugin     ‚Üí GET /health
  ‚îî‚îÄ‚îÄ api plugin        ‚Üí GET /api/v1/status, POST /api/v1/rebalance
         ‚îÇ
         ‚ñº
  RebalanceService (orchestrator)
  ‚îÇ       ‚îÇ          ‚îÇ           ‚îÇ
  ‚ñº       ‚ñº          ‚ñº           ‚ñº
ChainReader  Strategy  Executor  Notifier
(reads)      (pure)    (writes)  (Telegram)
```

**Key rules:**
- Strategy is pure ‚Äî no async, no RPC, no side effects
- Executor implements an interface ‚Äî swappable for dry-run mock
- Notifier is fire-and-forget ‚Äî never blocks rebalancing
- Error in one layer never crashes the bot process

### Hybrid IRM
- Read IRM parameters on-chain once per cycle (base rate, slopes, optimal utilization, jump multiplier)
- Compute projected post-rebalance APY off-chain in TypeScript
- Fewer RPC calls than pure on-chain; stays in sync unlike pure off-chain

## Domain Types

> No database schema ‚Äî all state is on-chain or in-memory. These are the TypeScript domain types.

### AdapterState
| Field | Type | Source |
|---|---|---|
| `address` | `Address` | Vault contract ‚Äî enabled adapters list |
| `adapterType` | `"morpho-market-v1" \| "morpho-vault-v1"` | Derived from adapter contract |
| `realAssets` | `bigint` | `adapter.realAssets()` |
| `allocationPercentage` | `number` | Computed: `realAssets / totalAssets` |
| `absoluteCap` | `bigint` | `vault.caps(riskId).absoluteCap` |
| `relativeCap` | `number` | `vault.caps(riskId).relativeCap` (basis points) |

### MarketData
| Field | Type | Source |
|---|---|---|
| `marketId` | `Hash` | Morpho Blue market identifier |
| `totalSupply` | `bigint` | `morpho.market().totalSupplyAssets` |
| `totalBorrow` | `bigint` | `morpho.market().totalBorrowAssets` |
| `availableLiquidity` | `bigint` | `totalSupply - totalBorrow` |
| `utilization` | `number` | `totalBorrow / totalSupply` |
| `currentSupplyAPY` | `number` | Derived from IRM |
| `irmParams` | `IRMParams` | Read from IRM contract |

### IRMParams
| Field | Type | Source |
|---|---|---|
| `baseRate` | `bigint` | IRM contract |
| `slope1` | `bigint` | IRM contract (rate below optimal utilization) |
| `slope2` | `bigint` | IRM contract (rate above optimal utilization) |
| `optimalUtilization` | `bigint` | IRM contract (kink point) |

### RebalanceAction
| Field | Type | Description |
|---|---|---|
| `adapter` | `Address` | Target adapter address |
| `direction` | `"allocate" \| "deallocate"` | Deposit to or withdraw from adapter |
| `amount` | `bigint` | Asset amount to move |
| `data` | `Hex` | ABI-encoded MarketParams (for market adapters) or `0x` (for vault adapters) |

### StrategyConfig
| Field | Type | Default | Source |
|---|---|---|---|
| `driftThresholdBps` | `number` | 500 (5%) | `DRIFT_THRESHOLD_BPS` env var |
| `gasCeilingGwei` | `number` | 50 | `GAS_CEILING_GWEI` env var |
| `maxMarketConcentrationPct` | `number` | 10 | `MAX_MARKET_CONCENTRATION_PCT` env var |
| `minLiquidityMultiplier` | `number` | 2 | `MIN_LIQUIDITY_MULTIPLIER` env var |
| `dryRun` | `boolean` | false | `DRY_RUN` env var |

## API Contract

| Method | Path | Auth | Request | Response (success) | Response (error) |
|---|---|---|---|---|---|
| `GET` | `/health` | None | ‚Äî | `200 { status, lastCheck, lastRebalance, uptime }` | `503 { status: "degraded", ... }` |
| `GET` | `/api/v1/status` | None | ‚Äî | `200 { data: { adapters[], totalAssets, lastRebalance, gasPrice, gasCeiling } }` | `500 { data: null, error: "..." }` |
| `POST` | `/api/v1/rebalance` | None | ‚Äî | `200 { data: { actions[], txHashes[] } }` or `200 { data: { action: "none", reason: "..." } }` | `409 { data: null, error: "Rebalance cycle already running" }` |

## Morpho V2 Contract Interactions

### Vault V2 (target vault configured via `VAULT_ADDRESS`)
| Call | Type | Purpose |
|---|---|---|
| `totalAssets()` | read | Total assets in vault |
| `adapters()` / enumeration | read | List of enabled adapters |
| `caps(bytes32 id)` | read | Absolute + relative caps per risk ID |
| `allocate(address adapter, bytes data, uint256 assets, bytes4 selector)` | write | Push assets from idle pool to adapter |
| `deallocate(address adapter, bytes data, uint256 assets, bytes4 selector)` | write | Pull assets from adapter to idle pool |

### MorphoMarketV1AdapterV2
| Call | Type | Purpose |
|---|---|---|
| `realAssets()` | read | Current assets supplied through this adapter |
| Adapter identification | read | Determine adapter type and underlying market |

### Morpho Blue (underlying protocol)
| Call | Type | Purpose |
|---|---|---|
| `market(Id)` | read | Total supply, total borrow, utilization |
| IRM contract reads | read | Base rate, slope1, slope2, optimal utilization |
