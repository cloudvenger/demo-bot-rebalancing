# Technical Plan — Morpho V2 Rebalancing Bot

> Generated by `/plan`. All sections derived from SPEC.md + docs/rebalancing/architecture.md.
>
> **2026-04-07 — V2 model correction.** The original plan assumed N market adapters per vault. In Morpho Vault V2, a vault has **one adapter per protocol** (e.g. one `MorphoMarketV1AdapterV2`) that routes to many Morpho Blue markets via `MarketParams` encoded in the call data. Domain types, contract interactions, and the deployment script have been rewritten against the verified ABIs from [`morpho-org/vault-v2`](https://github.com/morpho-org/vault-v2). See **Group 8** in [task.md](task.md) for the migration backlog applied to the existing src/ code.

## Stack Decisions

| Concern | Choice | Rationale |
|---|---|---|
| Runtime | Bun | Native TypeScript, fast startup, built-in test runner compatibility |
| Framework | Fastify | Structured plugin system, JSON schema validation, Pino logging built-in |
| Chain interaction | viem | Type-safe, modern, Morpho SDK compatible, native EIP-1559 support |
| Morpho SDK | `@morpho-org/blue-sdk` + `@morpho-org/blue-sdk-viem` | Official Morpho TypeScript SDK for reading market/vault state |
| Scheduler | croner | Lightweight cron for Bun, no Redis dependency (unlike BullMQ) |
| Testing | Vitest + Anvil | Vitest for unit tests, Anvil for forked mainnet integration tests |
| Config validation | zod | Runtime schema validation, great TypeScript inference |
| Logging | Pino | Fastify-native, structured JSON, zero-overhead in production |
| Telegram | Direct Bot API via `fetch` | No SDK dependency; single `sendMessage` endpoint is all we need |
| Deployment script | Foundry (forge) | Industry standard for Solidity scripting, fork-test support |
| Database | None (v1) | On-chain is the data source; DB for analytics is a v2 candidate |

## Architecture Patterns

> Full diagrams and rationale: [docs/rebalancing/architecture.md](docs/rebalancing/architecture.md)

### Service-Oriented Architecture

```
Fastify (HTTP + Plugins)
  │
  ├── scheduler plugin  → cron trigger
  ├── health plugin     → GET /health
  └── api plugin        → GET /api/v1/status, POST /api/v1/rebalance
         │
         ▼
  RebalanceService (orchestrator)
  │       │          │           │
  ▼       ▼          ▼           ▼
ChainReader  Strategy  Executor  Notifier
(reads)      (pure)    (writes)  (Telegram)
```

**Key rules:**
- Strategy is pure — no async, no RPC, no side effects
- Executor implements an interface — swappable for dry-run mock
- Notifier is fire-and-forget — never blocks rebalancing
- Error in one layer never crashes the bot process

### Hybrid IRM
- Read IRM parameters on-chain once per cycle (base rate, slopes, optimal utilization, jump multiplier)
- Compute projected post-rebalance APY off-chain in TypeScript
- Fewer RPC calls than pure on-chain; stays in sync unlike pure off-chain

## Domain Types

> No database schema — all state is on-chain or in-memory. These are the TypeScript domain types after the V2 single-adapter migration.

### MarketParams
On-chain Morpho Blue struct, used as both the market identifier and the call-data payload for `vault.allocate / vault.deallocate`.
| Field | Type | Source |
|---|---|---|
| `loanToken` | `Address` | Bot config (`MANAGED_MARKETS_PATH`) — must equal the vault's underlying asset |
| `collateralToken` | `Address` | Bot config |
| `oracle` | `Address` | Bot config |
| `irm` | `Address` | Bot config — must equal `AdaptiveCurveIRM` (the adapter enforces this) |
| `lltv` | `bigint` | Bot config (WAD) |

### ManagedMarket
A market the operator has configured the bot to manage. The bot's market list comes from config — there is no per-vault enumeration in V2.
| Field | Type | Source |
|---|---|---|
| `label` | `string` | Bot config — e.g. `"USDC/WETH 86%"` |
| `marketParams` | `MarketParams` | Bot config |
| `marketId` | `Hash` | Derived: `keccak256(abi.encode(marketParams))` (Morpho Blue market id) |
| `capIds` | `Hash[]` | Derived once at startup: `adapter.ids(marketParams)` — every id must have `absoluteCap > 0` for `allocate` to succeed |

### MarketAllocationState
Per-market on-chain snapshot read each cycle.
| Field | Type | Source |
|---|---|---|
| `market` | `ManagedMarket` | From config |
| `allocation` | `bigint` | `vault.allocation(marketSpecificId)` where `marketSpecificId = keccak256(abi.encode("this/marketParams", adapter, marketParams))` |
| `allocationPercentage` | `number` | Computed: `allocation / totalAssets` |
| `caps` | `Array<{ id: Hash, absoluteCap: bigint, relativeCap: bigint }>` | One entry per id in `market.capIds`, populated via `vault.absoluteCap(id)` + `vault.relativeCap(id)`. **`relativeCap` is a uint256 in WAD (1e18 = 100%), not basis points** — see § Verified cap-id preimages. |

### VaultState
Aggregate snapshot captured by ChainReader at the start of each cycle.
| Field | Type | Source |
|---|---|---|
| `vaultAddress` | `Address` | Config |
| `adapterAddress` | `Address` | Config (verified enabled at startup via `adaptersAt(i)`) |
| `totalAssets` | `bigint` | `vault.totalAssets()` |
| `marketStates` | `MarketAllocationState[]` | One per managed market |
| `marketData` | `MarketData[]` | Same length as `marketStates` — Morpho Blue + IRM data per market |

### MarketData
On-chain Morpho Blue market state + derived APY.
| Field | Type | Source |
|---|---|---|
| `marketId` | `Hash` | Morpho Blue market identifier |
| `totalSupply` | `bigint` | `morpho.market(id).totalSupplyAssets` |
| `totalBorrow` | `bigint` | `morpho.market(id).totalBorrowAssets` |
| `availableLiquidity` | `bigint` | `totalSupply - totalBorrow` |
| `utilization` | `number` | `totalBorrow / totalSupply` |
| `currentSupplyAPY` | `number` | Derived from IRM |
| `irmParams` | `IRMParams` | Read from IRM contract |

### IRMParams
| Field | Type | Source |
|---|---|---|
| `baseRate` | `bigint` | IRM contract |
| `slope1` | `bigint` | IRM contract (rate below optimal utilization) |
| `slope2` | `bigint` | IRM contract (rate above optimal utilization) |
| `optimalUtilization` | `bigint` | IRM contract (kink point) |

### RebalanceAction
A single `vault.allocate` or `vault.deallocate` call. The `adapter` field is constant across every action in a cycle (the configured single adapter); only `data` (the encoded `MarketParams`) varies per market.
| Field | Type | Description |
|---|---|---|
| `adapter` | `Address` | Target adapter address — always the configured single adapter |
| `marketLabel` | `string` | Human-readable market label for logs/alerts |
| `direction` | `"allocate" \| "deallocate"` | `allocate` pushes assets from idle pool to the market; `deallocate` pulls them back |
| `amount` | `bigint` | Asset amount to move |
| `data` | `Hex` | ABI-encoded `MarketParams` for the target Morpho Blue market |

### StrategyConfig
| Field | Type | Default | Source |
|---|---|---|---|
| `driftThresholdBps` | `number` | 500 (5%) | `DRIFT_THRESHOLD_BPS` env var |
| `gasCeilingGwei` | `number` | 50 | `GAS_CEILING_GWEI` env var |
| `maxMarketConcentrationPct` | `number` | 10 | `MAX_MARKET_CONCENTRATION_PCT` env var |
| `minLiquidityMultiplier` | `number` | 2 | `MIN_LIQUIDITY_MULTIPLIER` env var |
| `dryRun` | `boolean` | false | `DRY_RUN` env var |

## Morpho V2 Contract Interactions

> All signatures below are verified against the source at [`morpho-org/vault-v2`](https://github.com/morpho-org/vault-v2). The original plan's signatures (`caps(id)`, `realAssets()` per adapter, 4-arg `allocate`) are wrong and have been removed.

### VaultV2 (target vault configured via `VAULT_ADDRESS`)
| Call | Type | Purpose |
|---|---|---|
| `totalAssets() returns (uint256)` | read | Total assets in vault |
| `adaptersLength() returns (uint256)` | read | Number of enabled adapters (used at startup to verify the configured adapter is present) |
| `adaptersAt(uint256) returns (address)` | read | Adapter at index — used to verify `ADAPTER_ADDRESS` is enabled |
| `absoluteCap(bytes32 id) returns (uint256)` | read | Absolute cap for an id |
| `relativeCap(bytes32 id) returns (uint256)` | read | Relative cap for an id (basis points) |
| `allocation(bytes32 id) returns (uint256)` | read | Current allocation accounted for `id` |
| `isAllocator(address) returns (bool)` | read | Sanity check at startup that the bot wallet has the allocator role |
| `allocate(address adapter, bytes data, uint256 assets)` | write (allocator) | Push assets from idle pool to a specific market on the adapter (3 args, no selector) |
| `deallocate(address adapter, bytes data, uint256 assets) returns (bytes32[])` | write (allocator) | Pull assets from a specific market on the adapter back to the idle pool |
| `multicall(bytes[] calldata data)` | write | Used by the deployment script to batch curator setup |

### VaultV2 — curator setup calls (only used by the deployment script)
All are timelocked, but at fresh deploy the timelocks default to 0 so they can be `submit`-then-call in a single multicall:
- `setCurator(address newCurator)` — owner-only, **not** timelocked
- `submit(bytes calldata data)` — queue a timelocked call
- `addAdapter(address adapter)` — register the adapter
- `increaseAbsoluteCap(bytes idData, uint256 newCap)` / `increaseRelativeCap(...)` — set caps for each id returned by `adapter.ids(marketParams)`
- `setIsAllocator(address account, bool newIsAllocator)` — grant the bot wallet the allocator role

### MorphoMarketV1AdapterV2 (single adapter per vault)
| Call | Type | Purpose |
|---|---|---|
| `ids(MarketParams calldata) returns (bytes32[])` | read | Returns the 3 cap ids the vault checks during `allocate` for this market |
| `allocation(MarketParams calldata) returns (uint256)` | read | Convenience view forwarding to `vault.allocation(marketSpecificId)` |
| `parentVault() returns (address)` | read | Used to verify the adapter belongs to the configured vault |
| `adapterId() returns (bytes32)` | read | The adapter-wide id, set in the constructor as `keccak256(abi.encode("this", address(this)))` |

### Verified cap-id preimages

Source: [`src/adapters/MorphoMarketV1AdapterV2.sol:78,267-273`](https://github.com/morpho-org/vault-v2/blob/main/src/adapters/MorphoMarketV1AdapterV2.sol#L267) and [`src/VaultV2.sol:526-528`](https://github.com/morpho-org/vault-v2/blob/main/src/VaultV2.sol#L526) (verified 2026-04-07).

The vault stores `caps[id]` keyed by `id = keccak256(idData)`. The curator must therefore pass the **exact preimage bytes** to `increaseAbsoluteCap(bytes idData, uint256)` so that the resulting hash matches one of the ids the adapter returns from `ids(marketParams)`. If the preimage is wrong, the cap exists for an unrelated hash and `vault.allocate(...)` reverts with `ZeroAbsoluteCap`.

| # | Cap id semantics | `idData` preimage (bytes the curator passes) | TypeScript construction |
|---|---|---|---|
| 0 | Adapter-wide — applies to every market the adapter routes to | `abi.encode("this", address(adapter))` | `encodeAbiParameters([{type:'string'},{type:'address'}], ["this", adapter])` |
| 1 | Collateral-token — applies to every market sharing the same collateral token | `abi.encode("collateralToken", marketParams.collateralToken)` | `encodeAbiParameters([{type:'string'},{type:'address'}], ["collateralToken", collateralToken])` |
| 2 | Market-specific — applies to exactly one Morpho Blue market | `abi.encode("this/marketParams", address(adapter), marketParams)` | `encodeAbiParameters([{type:'string'},{type:'address'},{type:'tuple', components: marketParamsComponents}], ["this/marketParams", adapter, marketParams])` |

The bot computes the market-specific id #2 itself in TypeScript and asserts at startup that it matches `adapter.ids(marketParams)[2]`. If the assertion fails, abi encoding is wrong and the bot must abort.

### Cap units — `absoluteCap` and `relativeCap`

Verified at [`src/VaultV2.sol:592-595`](https://github.com/morpho-org/vault-v2/blob/main/src/VaultV2.sol#L592):
```solidity
require(_caps.absoluteCap > 0, ErrorsLib.ZeroAbsoluteCap());
require(_caps.allocation <= _caps.absoluteCap, ErrorsLib.AbsoluteCapExceeded());
require(
    _caps.relativeCap == WAD || _caps.allocation <= firstTotalAssets.mulDivDown(_caps.relativeCap, WAD),
    ErrorsLib.RelativeCapExceeded()
);
```

| Field | Unit | Notes |
|---|---|---|
| `absoluteCap` | uint256 in the vault asset's native decimals (e.g., USDC: 6 decimals) | **Must be > 0 for every id** the adapter returns, otherwise `allocate` reverts. Stored on-chain as `uint128` (`uint256` in the API for headroom). |
| `relativeCap` | uint256 in **WAD** (`1e18` = 100%). Example: 50% = `5e17`. | **NOT basis points.** The sentinel `relativeCap == WAD` means "no relative cap" and short-circuits the check. `relativeCap == 0` means `allocation <= 0` — effectively forbids the market. The bot must default new managed markets to a real WAD value (e.g. `WAD` for "unlimited" or `5e17` for 50%), never 0. |

> **No `realAssets()` per market — the bot reads `vault.allocation(id)` instead.**

### Group 8 — V2 single-adapter migration (sequential, after Group 7)
> Created 2026-04-07 after verifying the real V2 ABIs against [`morpho-org/vault-v2`](https://github.com/morpho-org/vault-v2). This group migrates the bot to the correct model.

See [task.md](task.md#group-8--v2-single-adapter-migration) for the full task list. Summary of work:
